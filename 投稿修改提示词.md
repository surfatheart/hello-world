# 论文投稿修改提示词（面向中文核心期刊）

> 用途：让 AI 助手按核心期刊标准修改论文，规避「AI 味、过度肯定、图表缺失、参考文献造假」四类常见问题。
> 用法：复制下方【提示词模板】，填好 `目标期刊` 与 `论文名`，连同「投喂清单」里的材料一起发给 AI。

---

## 一、提示词模板（直接复制使用）

```
【角色】你是中医药信息学/医学人工智能方向的资深论文作者与审稿人，熟悉中文核心期刊（目标期刊：____）的体例与审稿尺度。

【任务】对我提供的论文《____》进行面向该期刊投稿的修改，输出可直接投稿的版本，并附修改说明。

【写作风格（硬性）】
1. 像人写的中文学术语体，不要 AI 味：少用「然而/此外/因此/综上所述/值得注意的是/不仅……而且」等套话连接词；不排比堆砌；不堆形容词。
2. 避免过度肯定：结论用与证据强度匹配的措辞（可表明 / 提示 / 可能 / 有待验证）；凡「显著/首次/最佳/必将」等词须有数据或文献支撑，否则降级或删除。
3. 一句话一般只挂 1 个参考文献，最多 2 个；不要在概述句后堆一串引用。
4. 段落用「观点—证据—简短引申」结构，先给判断再给依据。

【图表（硬性）】
1. 至少包含：1 张流程/框架图（可用 draw.io 绘制并给出 .drawio 源文件）+ 2 张数据/分类表。
2. 每个图表必须在正文被引用（见图 X / 见表 X）；图题在下、表题在上。
3. 表格用三线表；图优先矢量（SVG/PDF），并提供可编辑源文件。

【参考文献（硬性，最重要）】
1. 只引用真实存在、可回溯的文献：每条须能由 DOI 或「期刊+年+卷(期):页」唯一定位。
2. 严禁编造文献、作者、DOI、卷期页；凡不确定的著录项，标记为「【待核】」，不要猜。
3. 统一 GB/T 7714 格式，按正文出现顺序从 [1] 编号；未被正文引用的不列入。
4. 优先使用我提供的检索题录（WoS/PubMed/CNKI 导出）核对著录信息。

【输出】
1. 先给「修改要点清单」（改了什么、为什么）；
2. 再给全文；
3. 最后给「存疑/待核清单」（哪些数据或引用需我确认）。

【约束】不得编造数据、实验结果、统计量或文献；不确定一律标注，宁缺毋造。
```

---

## 二、为什么这样写更有效

1. **把「去 AI 味」拆成可执行规则**：列出要少用的连接词、禁止排比、限制每句引用数。笼统说「别像 AI」没用，列黑名单才管用。
2. **把「别太肯定」变成「措辞 ≤ 证据」**：给对冲词梯度（可表明 ＞ 提示 ＞ 可能 ＞ 有待验证），并要求「显著/首次/最佳」必须有出处——这正是审稿最常挑的点。
3. **参考文献「可回溯」写成强制条款**：
   - 每条能用 DOI 或「卷(期):页」唯一定位；
   - 不确定就标【待核】、禁止猜——防 AI 编造文献（最致命硬伤）的关键；
   - 让 AI 用你提供的题录核对，比「凭记忆」可靠得多。
4. **图表写成「数量 + 引用 + 格式」三件套**：规定最少数量、必须被正文引用、三线表/矢量图 + 源文件。
5. **要求「修改清单 + 待核清单」**：便于快速 review，并把不确定处显式暴露，而非混进正文。

---

## 三、每次附带的「投喂清单」

提示词再好，没有真实材料仍可能被编造。每次修改时建议同时提供：

- 目标期刊名 + 其《投稿须知》（字数、栏目、参考文献格式、图表要求）；
- 论文当前稿（.docx）；
- **文献题录文件**（WoS/CNKI/PubMed 导出，含 DOI/卷期页）——参考文献「真实可回溯」的前提；
- 已确认的真实数据（如 PRISMA 各阶段实际筛选数），避免使用占位值。

---

## 四、收尾自检清单（投稿前过一遍）

- [ ] 摘要含「目的/方法/结果/结论」，方法写清检索库与检索日期。
- [ ] 全文无「然而/此外/综上所述」等套话扎堆；无空泛排比。
- [ ] 结论措辞与证据强度匹配；「显著/首次/最佳」均有出处。
- [ ] 每句引用 ≤ 2 条；概述句不堆引用。
- [ ] 图表数量达标、均被正文引用；三线表；图有可编辑源文件。
- [ ] 参考文献按 GB/T 7714、按出现顺序 [1] 起连续；正文编号与文献表一一对应。
- [ ] 每条文献可由 DOI 或卷(期):页 核实；【待核】项已全部落实。
- [ ] 未被引用的文献已删除；无编造数据与文献。
